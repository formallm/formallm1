Problem

Formal Mathematics for LLMs

Translate natural language math problems into Lean/Litex proof scripts that compile in a standard environment, with semantics consistent with the original statement.

  • Input: Natural language math statements
  • Output: Lean/Litex proof code (compiles successfully)
  • Focus: Formalization, proof generation, code correctness and verifiability
  • Scoring: Compile-first, then semantic consistency. Both must pass.

More: Litex · Lean

Background

While LLMs excel at natural language problem solving, their outputs are hard to verify and may hallucinate. Formalized mathematics enables precise, machine-checkable proofs (e.g., with Lean). Litex is a concise formal language designed for "coded reasoning" with intuitive syntax.

Challenges include formalizing concepts, accurate usage of Lean/Litex syntax and libraries, handling induction and quantifiers, choosing tactics, and avoiding compile-time errors (type mismatches, undefined symbols, dependency cycles).

Tasks

(1) Formal Translation

  • Translate natural language statements into Lean/Litex propositions.
  • Define mathematical objects properly and reuse library definitions.
  • Handle implicit assumptions, boundary conditions, and quantifiers.

(2) Proof Generation

  • Generate complete proof scripts to solve the problem.
  • Leverage existing lemmas and automation when appropriate.
  • Ensure code compiles in standard Lean/Litex environments.

(3) Automated Evaluation

DimensionDescription
CompilationDoes the Lean/Litex code compile without errors?
Semantic ConsistencyIs the formal proposition consistent with the natural language statement?

First check compilation; if passed, check semantic consistency. Both must pass to be correct.

Note: The submission must be a complete Lean/Litex file that compiles in one pass.

(4) Resources

Litex: https://litexlang.com/

Lean: https://lean-lang.org/
