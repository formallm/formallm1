Lean/Litex-only

  • All answers must be Lean/Litex code; no natural language explanations.
  • Submissions must compile successfully in a standard Lean/Litex environment.
  • Formal propositions must be semantically consistent with the original problems.

Evaluation

Submission

  1. Proof script: Lean/Litex code that compiles in one pass
  2. Semantic equivalence: Formal vs. natural language statement
  3. README: Environment and compile instructions
  4. Licenses: Code and data usage terms

Defense

  • Presentation: 15 min technical talk
  • Q&A: 10 min expert questions
  • Demo: Live system demo recommended
  • Materials: Slides, demo, appendix

Scoring (Compile-first)

  1. Compilation: Does the Lean/Litex code compile?
  2. Semantic consistency: Is the formal statement consistent with the problem?
  3. Both required; otherwise marked incorrect
